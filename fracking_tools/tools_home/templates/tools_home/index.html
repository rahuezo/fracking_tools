Calculating the Content Similarity of 2 Documents

Vector Space Model (VSM)
In information retrieval or text mining, the term frequency-inverse document frequency (tf-idf), is a well known method to determine how important a word is in a document.

VSM is an algebraic model representing textual information as a vector. The components of this vector in our case represent the importance and presence (Bag of Words) of a term in a document (i.e. tf-idf). VSM represents the features extracted from the document.

Cosine Similarity
Since documents can be modeled as vectors of term frequencies (i.e. numbers) we can perform math operations on them.

A well-known similarity measure for vectors is called the cosine similarity. Cosine similarity emerges from the dot product of two vectors.



We can rearrange this equation so that the cosine part is left isolated



The cosine term has a range from -1 to +1 that depends on the value of θ (the angle between the two vectors). If the angle between the two vectors is small (close to 0) then the cosine of this angle is at its maximum at +1 meaning the two vectors are pointing in the same direction. If the angle is 90°, then cosine is 0 meaning the vectors are perpendicular to each other. If the between the vectors is 180°, then the cosine is -1 meaning the vectors are pointing in opposite directions.

If the vectors we're working with are document vectors, having a cosine of +1 means the documents are talking about the same topic. If their cosine is 0, then the documents aren't talking about the same topic at all. Finally, if the cosine is -1, the documents are talking about completely opposite topics.

Calculating the Textual Similarity of 2 Documents

Near Duplicate Detection
To detect whether two documents are almost identical, first we reduce each document to a set of strings and then use Jaccard similarity to calculate the similarity of those sets.

To reduce a document to a set of strings, a technique called shingling is used. Shingling extracts the set of all substrings of length k (k-shingles) from a text. This is implemented by sliding a window of size k over the input document character by character and throwing the substrings into a set. For example, the document "abcabcac" yields the 3-shingles {"abc", "bca", "cab", "cac"}. This set eliminates duplicate shingles.
